# Phase 6 任务清单（Linear 化改造）

基于 `docs/prd/phase6-ideas.md` 拆解。  
本阶段重点不是新增业务能力，而是把现有前端收敛成更接近 Linear 的工作台体验。

**图例**：`[ ]` 未开始 | `[~]` 进行中 | `[x]` 已完成

---

## 0. 前置约束

- 沿用现有前后端接口与数据模型，不以新增后端字段为前提
- 沿用 Phase 4/5 已有 `CustomSelect`、`CustomDatePicker`、`CommandPalette`、`overlayStore`
- 本阶段优先处理视觉系统、信息密度、层级秩序、Issue workspace 心智
- 除非明确必要，不并行引入评论系统、活动流后端化、复杂批量操作等新能力

## 完成标准

- [x] 全局 token、灰阶、层级、圆角、边框和交互状态形成统一系统
- [x] 顶部工具栏从“筛选表单行”收敛为更低噪音的命令带
- [x] List 成为真正的高密度默认工作界面
- [x] Board 与 List 保持同一语义和系统感，而不是两套组件
- [x] Issue drawer 更像工作上下文，而不是管理后台表单
- [x] 首页整体第一感受更接近 Linear 风格的 issue workflow 工具

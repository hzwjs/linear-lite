# Frontend Rendering Lessons

## Preserve Stable Content Structure

当页面已经有稳定的数据语义和排版结构时，不要为了“加载态”去改写内容本身的文档结构。更稳妥的做法是：

- 已保存内容继续走原生渲染路径，保持原有 DOM 语义、换行关系和布局约束不变。
- 加载态只附着在表现层，例如给现有元素增加临时样式、骨架视觉或渐进增强标记。
- 只有“临时态内容”才使用自定义节点或占位包装，比如上传中、失败重试、本地预览。

经验总结：前端处理异步资源时，优先保证“结构稳定”，再优化“视觉反馈”。如果 loading 方案会改变内容流、段落关系或编辑光标语义，通常就是过度设计了。

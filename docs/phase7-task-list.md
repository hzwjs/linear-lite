# Phase 7 任务清单（子任务 Sub-issues 能力）

基于 `docs/prd/phase7-ideas.md` 与 `docs/linear-subtasks-analysis.md` 拆解。  
本阶段为 linear-lite 引入父子任务能力：数据模型、列表展示、详情 Sub-issues 区块与内联创建、多级支持与展示选项。

**图例**：`[ ]` 未开始 | `[~]` 进行中 | `[x]` 已完成

---

## 0. 前置约束

- 沿用 Phase 5/6 的视图模型、TaskEditor、TaskListView、BoardView、IssueComposer 等，不重做主流程
- 子任务与父任务共用同一张 `tasks` 表，通过 `parent_id` 区分；子任务即「有父任务的任务」
- 本阶段不实现：评论/选区转子任务、批量 Set/Remove parent、父/子状态联动、列表筛选「仅父/仅子」

---

## 完成标准

- [ ] 任务表具备 `parent_id`，API 与前端类型支持父子及子任务计数
- [ ] 列表支持显示/隐藏子任务、嵌套展示，子任务行缩进与父任务 x/y 正确
- [ ] 父任务详情内 Sub-issues 为真实列表、可内联创建、可折叠；子任务详情有父任务链接并可再建子任务
- [ ] 多级子任务在详情与列表中均可选「仅直接」或「展开多级」，计数语义一致
- [ ] 与 Phase 5/6 视图与交互兼容，无回归

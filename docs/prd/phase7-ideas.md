# Linear Lite - Phase 7 子任务（Sub-issues）能力

## 背景

基于 [linear-subtasks-analysis.md](../linear-subtasks-analysis.md) 对 Linear 子任务功能的实际操作分析，当前 linear-lite 在子任务方面仅有 UI 占位：

- **TaskEditor**：有「Sub-issues」区块与占位文案「No sub-issues. Add one to break down this task.」，无数据与创建逻辑。
- **TaskListView**：行内「Add sub-issue」按钮无绑定逻辑，点击无效果。
- **后端**：`tasks` 表无 `parent_id`，无父子关系；前端类型与 API 也未包含父子语义。

Phase 7 的目标是在不追求与 Linear 像素级一致的前提下，补齐**可用的父子任务能力**：支持将大任务拆成子任务、在列表与详情中正确展示、在父任务内创建子任务，并可选支持多级（子任务再挂子任务）及列表展示开关。

---

## 一、与 Linear 的对照（分析结论摘要）

| 能力 | Linear 行为 | Phase 7 目标 |
|------|-------------|--------------|
| 数据模型 | Issue 有 parent，子任务为独立 issue | 任务表增加 `parent_id`，子任务即普通任务挂父 |
| 列表展示 | Show sub-issues 控制是否显示子任务行；缩进 + 标题路径「标题 > 父任务」+ 父任务 x/y | 列表支持「显示/隐藏子任务」、缩进与父任务计数 |
| 详情 Sub-issues 区块 | 可折叠、Display options、Create new sub-issue、树形/仅直接子任务 | 父任务详情内可折叠子任务列表、内联创建、可选多级展示 |
| 新建子任务 | 内联表单：标题、描述、状态/优先级/负责人/标签、Discard/Create | 父任务内内联创建，必填标题，可选描述与属性 |
| 子任务详情 | 面包屑 + 父任务链接，可再建子任务 | 子任务详情展示父任务链接，支持再挂子任务（多级） |
| 多级 | Nested sub-issues 控制详情内是否展开子孙级；列表有 Nested 选项 | 支持多级；详情与列表均可选「仅直接子任务」或「展开多级」 |

---

## 二、Phase 7 的设计目标

1. **数据与 API**：引入父子关系（`parent_id`），创建/更新/列表/详情接口支持父子语义；删除或解绑时策略明确（如父删除时子任务置空 parent 或级联由产品决定）。
2. **列表**：在现有 List（及可选 Board）上支持「显示子任务」开关；开启时子任务独立成行、缩进、标题区可带父任务信息，父任务行显示子任务完成数/总数（x/y）。
3. **详情**：父任务详情内 Sub-issues 区块从占位变为真实列表；支持内联创建子任务（标题必填，描述与属性可选）；区块可折叠；可选「仅直接子任务」与「展开多级」展示。
4. **子任务详情**：打开子任务时展示父任务链接（及可选面包屑）；子任务自身也可作为父任务再添加子任务（多级）。
5. **视图状态**：列表的「显示子任务」「嵌套子任务」等选项需持久化（视图级或用户级），与现有 view state 统一。

非目标（本阶段不实现）：从评论/描述选区转成子任务、Set parent / Remove parent 的批量操作、父/子状态联动（如全部子任务完成自动关父）、列表筛选「仅父/仅子」、子任务层级深度限制。

---

## 三、范围边界

- **后端**：仅增加 `parent_id` 及配套索引、约束；不新增「子任务」专用接口，在现有 task CRUD 与列表接口中扩展。
- **前端**：复用现有 TaskEditor、TaskListView、BoardView、view state；不重做创建/打开主流程，只在现有流程上叠加父子能力。
- **多级**：允许子任务再挂子任务；展示上提供「仅直接子任务」与「展开多级」选项，与 Linear 的 Nested sub-issues 语义对齐。
- **兼容**：无 `parent_id` 的任务视为顶层；现有任务数据无需迁移即可作为「仅顶层」列表使用。

---

## 四、验收标准

- 可为任意任务添加子任务（含多级）；子任务在列表中按开关显示、有缩进与父任务信息、父任务行有 x/y。
- 在父任务详情内能看到真实子任务列表、可内联创建、可点击进入子任务；子任务详情能回跳父任务并支持再建子任务。
- 列表的「显示子任务」「嵌套子任务」选项生效且可持久化；与 Phase 5/6 的视图系统一致，不破坏现有筛选、分组、排序。

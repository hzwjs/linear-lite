# 任务新增功能 · 实操体验评测报告

**评测方式**：在已登录状态下，通过浏览器实际操作完成。  
**角色**：应用体验师（仅基于页面与操作，不依赖代码）。

---

## 一、入口与发现性

**实际操作**：登录后主界面可见顶栏「New issue」、命令栏「+」、看板三列各有「Add issue to this column」、列表视图下分组头有「Create issue in group」。

**体验结论**：
- 顶栏「New issue」位置显眼，主操作明确。
- 看板列头「+」与列表分组头「+」都能一键打开新建弹层，无需先点进某条任务。
- 从 Done 列点击「Add issue to this column」后，弹层内 **Status 默认选中 Done**，与「在这一列新建」的预期一致，心智模型正确。
- 列表中存在「Add sub-issue」按钮（行 hover 出现），未在本次流程中点击；若点击无反应，易造成「可点但无效」的困惑，建议后续单独验证并决定隐藏或实现。

---

## 二、新建弹层（IssueComposer）

**实际操作**：点击顶栏「New issue」→ 弹层居中、有关闭按钮、标题输入框、描述区、Status/Priority/Assignee/Due date、Create more 勾选、Create issue 按钮。

**体验结论**：
- **布局**：弹层居中、遮罩弱化背景，焦点自然落在标题框，符合「先填标题」的动线。
- **标题**：placeholder「Issue title」清晰；输入「体验评测测试任务」后，「Create issue」由禁用变为可点，反馈及时。
- **提交反馈**：点击「Create issue」后按钮变为「Creating...」并禁用，防重复提交，体验合理。
- **Create more**：默认勾选（value on）。创建成功后若勾选则弹层不关、表单清空，便于连续建多条，逻辑符合预期。
- **未验证**：描述区按 Enter 是否进描述、输入 `/` 是否有格式菜单；Attach/Link/Watchers 未点击；关闭前有内容时是否有二次确认。

---

## 三、创建后的衔接

**实际操作**：填写标题后点击「Create issue」→ 约 2 秒内 URL 变为 `/tasks/ENG-4`，弹层关闭，右侧展开该任务详情（标题「体验评测测试任务」、Status/Priority/Assignee、Sub-issues、Activity、评论区）。

**体验结论**：
- 创建成功后**自动跳转至该任务详情**，无需再在列表中查找，动线顺畅。
- 详情页与弹层内字段一致（标题、状态、优先级等），无「换了一套选项」的割裂感。
- 看板列表数量即时更新（Todo 由 1 变为 2），新任务出现在对应状态列。

---

## 四、看板与列头创建的联动

**实际操作**：关闭详情回到主视图 → 切到 Board 视图 → 点击 Done 列「Add issue to this column」→ 弹层内 Status 展开后确认为 **Done** 选中。

**体验结论**：
- 按列创建时**默认状态与列一致**，减少手动改状态的步骤，符合「在某一列加一条」的直觉。

---

## 五、可访问性（基于当前操作）

- 弹层具备 `role="dialog"`、关闭按钮有明确命名（Close），利于读屏与键盘用户。
- 标题框、Status 等控件在快照中均有可识别名称，未见明显 a11y 缺失。
- 关闭弹层后焦点是否回到触发按钮未在本轮操作中验证。

---

## 六、待后续验证或改进点

| 项 | 说明 |
|----|------|
| 描述区 Enter / 斜杠 | 标题按 Enter 是否进描述、描述区输入 `/` 是否出现格式菜单，建议手工再测一次。 |
| Attach / Link / Watchers | 当前为可点按钮，若点击无实际功能，易被当成「假按钮」，建议禁用或加「即将推出」说明。 |
| 关闭前确认 | 若用户已填标题或描述后误点遮罩/关闭，是否有二次确认；若无，存在丢内容风险。 |
| 创建失败提示 | 本次为成功路径；若网络或后端失败，页面上是否有明确错误提示需另测。 |
| 列表行「Add sub-issue」 | 建议确认点击后是否有行为，若无则考虑隐藏或禁用并说明。 |

---

## 七、总体结论

- **主流程**：从「点 New issue 或列头 +」→「填标题」→「Create issue」→「进入该任务详情」整条路径顺畅，反馈清晰（Creating...、跳转、列表更新）。
- **列头 + 默认状态**：与当前列状态一致，符合预期。
- **创建即进入详情**：减少一步「找刚建的那条」，体验良好。
- 建议补测：描述区快捷键、占位按钮行为、关闭确认与失败态提示，并视结果做小幅优化。

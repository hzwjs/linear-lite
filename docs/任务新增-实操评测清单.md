## 二、找到「新建任务」入口

| 步骤 | 操作 | 你的观察 |
|------|------|----------|
| 2.1 | 看顶栏：有没有「New issue」按钮？是否显眼？ | |
| 2.2 | 看命令栏（All issues 那一行）：有没有「+」？点一下会怎样？ | |
| 2.3 | 若在看板视图：每个列头有没有「+」？点某一列的「+」 | 弹层里默认状态是否对应该列（如 Todo 列的 + 默认 Todo）？ |

## 三、新建弹层（IssueComposer）操作

| 步骤 | 操作 | 你的观察 |
|------|------|----------|
| 3.1 | 任选一入口打开新建弹层 | 弹层是否居中、遮罩是否让背景弱化、是否一眼知道要填什么？ |
| 3.2 | 看标题：是否写「New issue」？标题输入框是否自动聚焦？ | |
| 3.3 | 输入任务标题，按 Enter | 焦点是否跳到描述区？ |
| 3.4 | 在描述区输入几行字，或试一下输入 `/` | 是否有格式提示/菜单？ |
| 3.5 | 看 Status / Priority / Assignee / Due date 四个控件 | 是否容易点、选项是否清晰、默认值是否合理？ |
| 3.6 | 不填标题，直接点「Create issue」 | 按钮是否不可点或有点击无反应？ |
| 3.7 | 填好标题后点「Create issue」 | 是否有「Creating...」等反馈？创建成功后弹层是关闭还是留在原地？是否跳转到该任务详情？ |
| 3.8 | 勾选「Create more」再创建一条 | 创建成功后表单是否清空、弹层是否仍打开？ |
| 3.9 | 填了标题和描述后，点遮罩或右上角 × 关闭 | 是否有二次确认？若没有，是否担心误关丢内容？ |
| 3.10 | 弹层内 Attach / Link / Watchers 按钮 | 点一下是否有反应？若没有，是否觉得像「假按钮」？ |
